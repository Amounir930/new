# دليل التعامل مع Gitea (git.60sec.shop)

هذا المستند يوضح كيفية التعامل مع مستودع الكود الخاص بمشروع **Apex v2** بشكل آمن واحترافي.

---

## 1. معلومات الوصول الأساسية
- **الرابط:** [https://git.60sec.shop](https://git.60sec.shop)
- **المستودع:** `60sec.shop/apex-v2`
- **البروتوكول المفضل:** HTTPS (للسهولة والاستقرار).

---

## 2. إعداد الاتصال المحلي (Workstation)
لربط جهازك المحلي بـ Gitea، قم بتنفيذ الأمر التالي في مجلد المشروع:

```bash
git remote add gitea https://git.60sec.shop/60sec.shop/apex-v2.git
```

---

## 3. دورة التطوير والنشر (Workflow)
بناءً على القواعد الصارمة للمشروع، اتبع هذا التسلسل:

### أ. فحص الكود محلياً
تأكد من عدم وجود أخطاء تنسيق (Lint) أو ثغرات أمنية (Security):
```bash
bun lint
bun audit
```

### ب. دفع التعديلات
يفضل دائماً رفع التعديلات من فرعك الحالي (مثلاً `New`) إلى الفرع الرئيسي في Gitea:
```bash
git push gitea New:main
```

---

## 4. النشر التلقائي (Webhooks)
المشروع مزود بـ **Webhook Listener** يقوم بتحديث السيرفر تلقائياً عند أي `push` فرع `main`.

> [!IMPORTANT]
> إذا فشل النشر التلقائي (مثلاً بسبب مشاكل في التوقيع الرقمي أو الشبكة)، يمكنك دائماً النشر يدوياً من داخل السيرفر.

---

## 5. النشر اليدوي (Manual Deploy)
في حال أردت تحديث السيرفر يدوياً:
1. اتصل بالسيرفر عبر SSH.
2. توجه لمجلد النشر: `cd ~/deployment`.
3. اسحب الكود من Gitea:
   ```bash
   git pull gitea main
   ```
4. أعد بناء الحاويات:
   ```bash
   docker compose -f ops/docker-compose.prod.yml up -d --build --no-deps api
   ```

---

## 6. الأمان (S1-S15)
- **Deployment Keys:** يتم استخدام مفاتيح SSH خاصة مخزنة في `~/.ssh/gitea_deploy_key` داخل السيرفر للسحب من Gitea.
- **Internal Network:** السيرفر يتصل بـ Gitea عبر شبكة Docker الداخلية لضمان السرعة والأمان.
